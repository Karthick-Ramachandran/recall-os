# Module: Hooks

## Purpose

Hooks owns generation of the repository's git pre-commit hook and the neutral detection of proposed
pre-commit gates.

It makes `persist doctor` and a repository's own gates enforceable at commit time without encoding a
toolchain into core.

## Owns

- The tracked pre-commit hook path (`.persist/hooks/pre-commit`).
- Rendering of the deterministic `sh` hook content.
- Neutral detection of proposed `preCommitGates` from a target repository's toolchain.
- The activation command text proposed to the user.

## Does Not Own

- The `preCommitGates` config field definition (owned by `config`).
- Writing files or setting the executable bit (owned by `filesystem`).
- Running `git config` or any git mutation.
- Doctor checks.
- Network, telemetry, MCP runtime, AI API, or cloud behavior.

## Public Interfaces

- `detectPreCommitGates`
- `renderPreCommitHook`
- `PRE_COMMIT_HOOK_PATH`
- `HOOKS_DIR`
- `HOOKS_PATH_ACTIVATION_COMMAND`

## Boundaries

Hooks produces content and proposed values. It does not write files, change git configuration, or
execute repository content. Detection only reads `package.json` and lockfiles.

## Current Decision

Governed by ADR-0002. The hook is tracked at `.persist/hooks/pre-commit`, runs `persist doctor` then
configured gates, and is generated by default during `persist init` while staying non-destructive.
Activation (`git config core.hooksPath .persist/hooks`) is proposed, never run.
