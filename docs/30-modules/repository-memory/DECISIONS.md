# Repository Memory Decisions

## P1.7: Repository Memory First

Recall OS is not a repo bootstrap tool or application generator.

Recall OS initializes and protects repository memory.

## P1.7: Detection Is Not Acceptance

Technology detection may inform suggestions.

Technology detection must not become accepted repository memory by itself.

## P5: Minimal Runtime Memory

`recall init` now creates minimal durable repository memory.

Generated docs are concise placeholders meant to route humans and agents to source-of-truth files.

## P11: Memory Before Automation

Recall OS should prioritize memory protection and drift detection before broader automation.

Future adoption and organization memory should produce reviewable memory and must not silently
accept inferred decisions.
