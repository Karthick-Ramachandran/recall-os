# Acceptance Criteria: Product Vision Roadmap Alignment

## Product Memory

- `docs/00-product/PRODUCT_VISION.md` exists.
- `docs/00-product/POSITIONING.md` exists.
- `docs/00-product/ROADMAP.md` exists.
- Product docs state that Persist OS is an Engineering Memory Operating System.
- Product docs state the core promise: AI can write code, Persist OS keeps repository memory intact.
- Product docs include: Git tracks what changed; Persist OS tracks why it changed.

## Roadmap

- Roadmap records current completed foundation.
- Roadmap names the next product priority as drift detection.
- Roadmap includes future levels for AI context distribution, governance, legacy adoption, and
  organization memory.
- Roadmap keeps adoption and organization memory as future work.

## Anti-Scope

- Product memory explicitly rejects becoming an AI coding agent, IDE competitor, app generator,
  architecture generator, agent runtime, model host, or cloud execution platform.

## Routing

- README links the core promise and product direction.
- `AGENTS.md` and `CLAUDE.md` route agents to product vision, positioning, and roadmap.
- Root and canonical product docs stay synchronized where relevant.

## Runtime Boundary

- No runtime code changes.
- No CLI behavior changes.
- No dependencies.
- No network, telemetry, MCP runtime, AI API, cloud behavior, or generated app code.
